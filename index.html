Brunch

A lightweight approach to building HTML5 applications with emphasis on elegance and simplicity.

Follow on GitHub

Application assembler

Brunch is an assembler for HTML5 applications. It's agnostic to frameworks, libraries, programming, stylesheet & templating languages and backend technology.

To simplify app development, brunch watches your files for changes and automatically wraps your scripts and templates in common.js modules. All build errors are printed to console. If you have growl / libnotify, brunch will use them too.

Brunch uses node.js plugins to speak with compilers. There are a bunch of plugins already:

  • Programming languages: JavaScript, CoffeeScript, IcedCoffeeScript, Roy
  • Pre-compiled templates: Eco, Jade, Mustache, Handlebars.js
  • Styles: CSS, Stylus, LESS, Sass
  • Minifiers: uglify.js, clean-css

In case your favorite language / tool is not supported yet, you can write brunch plugin for it, it's very simple. See documentation for required steps. Feel free to edit wiki page and add new plugins there.

Brunch doesn't encourage you to use any particular technology, but application skeletons can do it. Some interesting skeletons:

  • Base (comes with brunch): very simple, HTML5 Boilerplate with Backbone.js. Main languages are CoffeeScript, Stylus and Eco.
  • Brunch with Chaplin: Bootstrap, HTML5 Boilerplate, Chaplin classes over Backbone.js. Main languages are CoffeeScript, Stylus and Handlebars.
  • Jonssonlemacks skeleton: base skeleton with responsive Stylus grid, modernizr and more to come.
    • Of course, you can throw skeleton technologies away and buald your own structure.

    Getting started

    You'll need node.js 0.6.10+. Execute:

    npm install -g brunch

    You're done! Note that windows support is not there yet, but it's coming soon™.

    To create a new project, execute brunch new <project name>.

    Now you can edit the code. app/ directory has files of your project and vendor/ has libraries etc. that would not be wrapped in common.js modules.

    To build it, execute brunch build in project directory.

    To continuously rebuild the project on every change, run brunch watch.

    To run a simple http server, execute brunch watch --server.

    To install / uninstall a plugin, add it to package.json and execute npm install

    See docs for more info. If you're upgrading from earlier version, you might be interested in upgrading guide and changelog.

    Contributing to Brunch

    • Fork the official repository.
    • Clone your fork: git clone git@github.com:<your-username>/brunch.git
    • Make sure tests are passing for you: npm install && npm test
    • Create a topic branch: git checkout -b features/new-feature-name
    • Add tests and code for your changes.
    • Once you're done, make sure all tests still pass: npm install && npm test
    • Make sure your code conforms coffee style guide
    • Commit and push to your fork.
    • Create an pull request from your branch.
    • Sit back and enjoy.

    If the feature you want to add is large, you should probably ask brunch folks if it would be accepted. To do it, start new discussion on github or ping @brunch / @paulmillr on twitter.
